Accessibility

Syhrezz targets WCAG 2.1 AA compliance. This page documents the keyboard navigation, focus management, ARIA patterns, and contrast requirements for every component. Note: full validation requires manual testing with assistive technologies.

Keyboard Navigation

KeyAction
TabMove focus forward through interactive elements
Shift+TabMove focus backward
Enter / SpaceActivate buttons, checkboxes, open dropdowns
EscapeClose modals, dropdowns, search results
Arrow keysNavigate within dropdowns, tab lists, radio groups
⌘K / Ctrl+KOpen global search

Focus Management

ComponentFocus Rule
Modal openMove focus to first focusable element inside modal. Trap focus within modal while open.
Modal closeReturn focus to the trigger button that opened the modal.
Dropdown openMove focus to first menu item.
Sidebar collapseFocus does not change. Collapsed nav items remain focusable (icon-only).
Toast appearDo not steal focus. Toast is announced via aria-live region.

Focus Ring

All interactive elements show a visible focus ring. Syhrezz uses a 3px indigo ring with a slight offset:

:focus-visible {
  outline: 2px solid #6366F1;
  outline-offset: 2px;
}
/* For inputs/textareas, use box-shadow instead */
.input:focus {
  border-color: #3B5BDB;
  box-shadow: 0 0 0 3px rgba(59,91,219,0.1);
}

Touch Targets

All interactive elements meet the WCAG 2.5.5 minimum touch target size of 44×44px on mobile. Button default height is 44px. Nav items are at least 44px tall. Checkboxes use 16px with a label that extends the click area.

Full WCAG compliance requires manual testing. This documentation covers the design system's built-in accessibility features. Actual compliance depends on how components are implemented in context. Test with screen readers (NVDA, VoiceOver) and keyboard-only navigation before shipping.
