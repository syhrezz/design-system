Breakpoints

Syhrezz defines three responsive breakpoints. The layout transitions from a full desktop shell, to a tablet-optimised view, to a fully mobile layout with an off-canvas sidebar.

Breakpoint Scale

Desktop — > 1200px — Full sidebar + all grid columns + right TOC
Tablet — 768px–1200px — Sidebar visible, right TOC hidden, grids collapse
Mobile — < 768px — Off-canvas sidebar, single column, bottom nav

Mobile-Specific Patterns

On mobile, the sidebar becomes an off-canvas drawer. A hamburger button in the topbar toggles it. A dark overlay covers the main content when the sidebar is open. Tapping the overlay closes the sidebar.

/* Mobile sidebar — off canvas by default */
@media (max-width: 768px) {
  .ds-sidebar { transform: translateX(-100%); transition: var(--transition-sidebar); }
  .ds-sidebar.open { transform: translateX(0); }
  .ds-main { margin-left: 0; }
  .ds-topbar { left: 0; }
}

AI Implementation Notes

For AI coding assistants: Syhrezz has exactly 3 breakpoints: 1200px (tablet/desktop boundary) and 768px (mobile boundary). Breakpoints are defined in CSS @media queries only — not in JavaScript. The sidebar never disappears on tablet — it only goes off-canvas on mobile (<768px). Always use var(--transition-sidebar) for sidebar slide animations.
