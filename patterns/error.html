Error States

Error states communicate system failures, validation errors, and access issues. Syhrezz uses four error patterns: field validation, inline alert banners, error empty states, and full-page error screens.

Field Validation Errors

Validation errors appear below the field immediately. The field gets .input-error (red border + red focus ring). A .form-error-msg div shows below with an icon and message.

Please enter a valid email address

Alert Banner Errors

For non-field errors like server failures, permission issues, or multi-field validation, use an .alert.alert-danger banner at the top of the form or page section.

Failed to save changes. The server returned an error. Please check your connection and try again.

Error Empty State

When a section fails to load data, render an error empty state instead of the skeleton or content.

Failed to load data
Something went wrong while loading this section. This is usually temporary.

AI Implementation Notes

For AI coding assistants: Field errors: add .input-error to input + render .form-error-msg with icon below it. Section load errors: replace skeleton/content with error empty state (red icon bg #FDE2E2, stroke-width:1.5). Never show raw error messages to users — always provide a human-readable message and a Retry action. Error empty state icon background is #FDE2E2 with icon stroke #F05252 at stroke-width:1.5.
