Empty States

Empty states appear when a list, table, or section has no content. They guide users toward the next action and prevent the UI from feeling broken. Every list view must have an empty state.

Empty State Types

TypeWhen to UseIcon BGCTA
No data yetCollection is empty — user hasn't added anything#F1F5F9"+ Add First [Item]" .btn-primary
No resultsFilter/search returned 0 results#F1F5F9"Clear Filters" .btn-outline
Error stateData failed to load from server#FDE2E2"Retry" .btn-outline
PermissionUser doesn't have access to this section#FEF3C7"Request Access" or none

AI Implementation Notes

For AI coding assistants: Every list page and table MUST have an empty state. Use different messages for "no data yet" vs "no search results" — the CTA and tone differ. Icons use stroke-width:1.5 (thinner than UI icons at 2) for a lighter feel. Error empty states change icon bg to #FDE2E2 and icon color to #F05252. Never leave a table empty without an empty state — the UI must never look broken.
