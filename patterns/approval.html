Approval Flow

Approval flows handle multi-step confirmation patterns — from simple delete confirmations to multi-stage approval workflows. All destructive or consequential actions in Syhrezz require explicit confirmation.

Delete Confirmation

The simplest approval pattern. A .modal.modal-sm with a clear description of what will be deleted, a Cancel button, and a .btn-danger confirm button.

Delete User?

This will permanently delete Budi Santoso and all associated data including activities, prospects, and transactions. This action cannot be undone.

Status Approval Flow

Used for approving/rejecting items like announcements, leave requests, or budget approvals. Shows current status badge and provides Approve/Reject actions.

Rules

RuleDetail
Delete always confirmsEvery delete action MUST show a confirmation modal. Never fire delete directly from a button click.
Describe consequencesThe confirm modal must describe exactly what will be deleted/changed, including cascading effects.
Danger button placementDanger/destructive confirm button is always on the RIGHT. Cancel is always on the LEFT.
Approve/Reject orderApprove (green) always left of Reject (danger outline) in approval rows.
Status badgeEvery approval item shows its current status as a badge: Pending (warning), Approved (success), Rejected (danger).
Irreversible warningIf an action cannot be undone, state this explicitly in the modal body text.

AI Implementation Notes

For AI coding assistants: Delete confirmation uses .modal.modal-sm (max-width:380px). The confirm button is ALWAYS .btn-danger — never .btn-primary for destructive actions. Modal body must name the specific item being deleted and state if the action is irreversible. For approval queues, Approve uses .btn-success and Reject uses .btn-outline with red color override. Always show the current status as a badge on each row.
