Popover

Popover is a click-triggered floating panel that contains rich content — unlike Tooltip (hover-only, text-only), Popover supports interactive content including buttons, forms, and lists. It appears on click and closes when clicking outside.

When to Use

Use Popover when
  • Contextual info panel triggered by click (not hover)
  • Inline mini-forms (date picker, color picker trigger)
  • User profile preview card on hover/click
  • Rich help/documentation panel
  • Confirmation panel for non-destructive actions
Don't use Popover when
  • Simple text hint — use Tooltip instead
  • List of actions — use Dropdown instead
  • Complex full forms — use Modal instead
  • Critical blocking content — use Modal (popover can be dismissed)

Anatomy

1
Trigger
Any element. Button is most common. Must have aria-expanded and aria-controls attributes. Click toggles popover open/closed.
2
Panel
White card. bg:#fff, border:1px solid #E2E8F0, border-radius:14px, box-shadow (0 10px 40px rgba(0,0,0,.12)), padding:16px, min-width:220px. z-index:400.
3
Arrow
CSS border triangle via ::after and ::before pseudo-elements. Points toward trigger. Color matches panel bg and border.
4
Content
Free-form — title, description, buttons, forms. Not restricted like Tooltip.

Behavior

TriggerBehavior
Trigger clickToggles panel display:none → display:block. Adds document click listener for outside click detection.
Click outsideCloses panel. Use mousedown (not click) listener to prevent conflicts with trigger click.
Escape keyCloses panel. Returns focus to trigger button.
Tab out of panelCloses panel when focus moves outside the popover content.

Best Practices

Do
Keep popover content focused and minimal. One clear purpose per popover.
Popovers are lightweight — if content grows complex, promote to Modal.
Don't
Nest multiple popovers or open popovers from inside other popovers.
Layered overlays create confusing z-index conflicts and poor UX.

AI Implementation Guide

For AI coding assistants: Popover = Tooltip + rich content + click trigger. Container: position:relative on wrapper. Panel: position:absolute, bg:#fff, border:#E2E8F0, radius:14px, shadow, min-width:220px, z-index:400. Arrow: CSS border triangles via ::before/::after. Click outside: document.addEventListener("mousedown") — check if !wrap.contains(e.target). Escape closes. aria-expanded on trigger. Unlike Dropdown (action list), Popover contains informational or form content.

Design Decisions

How is Popover different from Dropdown?
Dropdown is a menu — a list of actions. Its items are navigation links or action buttons. Popover is an information panel — it can contain anything: text, forms, profile cards, status details. The distinction is purpose: Dropdown = "choose an action", Popover = "view information or interact with a mini-panel".
Why position top (above trigger) as default?
Popovers above the trigger don't obstruct the content the user just clicked. Most triggers are in toolbars or inline with content — positioning above reveals the contextual information without covering other relevant content below.
