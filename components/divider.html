Divider

Dividers are thin visual separators that create clear boundaries between content sections. They can be horizontal (between rows) or vertical (between columns), and optionally carry a text label.

When to Use

Use Divider when
  • Separating distinct groups within a card
  • Between form sections
  • Dropdown menu item groups
  • "or" separators between actions (e.g. login form)
Don't use Divider when
  • Spacing alone is sufficient — use margin/gap
  • Between every item — creates visual noise
  • Cards already provide visual separation

Anatomy

1
Line
1px solid #E2E8F0. For horizontal: border-top. For vertical: width:1px, background:#E2E8F0. Use <hr> for semantic horizontal dividers.
2
Label (optional)
Centered text between two lines. 12px fw-600 color:#94A3B8. Lines extend to fill remaining width via flex:1 pseudo-elements.

States

Dividers are purely presentational — no interactive states apply. They render statically and do not respond to any user interaction.

Behavior

Dividers are static. No behavior. They render as a visible line. Labeled dividers use CSS ::before and ::after pseudo-elements on the wrapper div to create the extending lines.

Accessibility

RequirementImplementation
Horizontal dividersUse <hr> element for semantic horizontal dividers. Adds role="separator" automatically.
Decorative dividersIf purely decorative (visual only), add aria-hidden="true".
Labeled dividersUse a div with the label text — no special ARIA needed.

Best Practices

Do
Use dividers sparingly — only when spacing alone isn't enough to signal a content boundary.
Too many dividers create visual noise and reduce readability.
Don't
Add dividers between every card, every section, and every item on a page.
Over-dividing fragments the layout and makes it harder to scan.

AI Implementation Guide

For AI coding assistants: Horizontal divider: <hr class="divider"> with border-top:1px solid #E2E8F0. Labeled divider: <div class="divider-label">or</div> with ::before/::after pseudo-elements (flex:1, border-top). Vertical: width:1px, background:#E2E8F0, align-self:stretch. Color is always #E2E8F0 — never change this.

Design Decisions

Why #E2E8F0 specifically?
This is the --border token — the same color used for card borders, input borders, and table row borders. Using the same color for dividers ensures visual consistency. A divider should be subtle — barely there — just enough to signal a boundary without drawing attention to itself.
