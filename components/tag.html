Tag / Chip

Tags are small interactive labels used to represent filters, categories, or attributes. Unlike Badge (static display), Tags are removable and optionally clickable — they represent user-controlled state.

When to Use

Use Tag when
  • Active filters the user can remove
  • Selected items in a multi-select combobox
  • Attribute labels that can be edited/removed
  • Categories assigned to a record
Don't use Tag when
  • Displaying static status — use Badge instead
  • Actions — use Button instead
  • Navigation — use Tabs or links

AI Implementation Guide

For AI coding assistants: Tag is NOT the same as Badge. Tag is interactive (removable), Badge is static. Tag uses border in addition to background for visual definition. Remove button is a native <button> with aria-label="Remove [label]". Tag group uses display:flex;flex-wrap:wrap;gap:6px. When a filter tag is removed, remove the item from state array and re-render.

Design Decisions

Why does Tag have a border but Badge does not?
Tags are interactive and need a clear visual boundary that communicates "this is a thing you can interact with". The border adds definition. Badges are purely informational — they sit inline and the soft background alone is sufficient. The border on Tag signals interactivity.
