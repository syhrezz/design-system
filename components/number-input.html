Number Input

Number Input extends the standard Input with increment/decrement stepper buttons. Used when users need to enter a precise numeric value with constrained min/max bounds, such as quantity, age, score, or page count.

When to Use

Use Number Input when
  • Entering a specific numeric value (quantity, count, score)
  • Bounded values where min/max matter
  • User benefits from increment/decrement control
Don't use Number Input when
  • Selecting a value from a range — use a Slider
  • Large numbers without bounds — use a plain Input type="number"
  • Choosing from discrete options — use Select or Radio

Properties

PropertyTypeRequiredDefaultDescription
valuenumberrequired—Controlled numeric value.
minnumberoptionalundefinedMinimum allowed value. Disables decrement button when reached.
maxnumberoptionalundefinedMaximum allowed value. Disables increment button when reached.
stepnumberoptional1Amount to increment/decrement per click.
onChangefunctionrequired—Called with new numeric value on every change.
disabledbooleanoptionalfalseDisables all interaction. opacity:.45.

Behavior

TriggerBehavior
Decrement clickvalue = Math.max(min, value - step). Calls onChange. Disables button if new value === min.
Increment clickvalue = Math.min(max, value + step). Calls onChange. Disables button if new value === max.
Direct inputUser can type directly. Clamp to min/max on blur. Reject non-numeric characters.
Arrow keysWhen input is focused, Up/Down arrows increment/decrement by step. Native behavior.

Best Practices

Do
Always show hint text with min/max bounds so users know the valid range.
Prevents frustration when values are silently clamped.
Don't
Use Number Input for values like price, phone number, or ID — use plain Input instead.
Increment/decrement is misleading for non-countable numeric fields.

AI Implementation Guide

For AI coding assistants: Three elements inside .number-input-wrap: button.left (decrement), input[type=number] (value), button.right (increment). Container has border + border-radius:10px + focus-within ring. Hide native number spinners: input::-webkit-inner-spin-button{appearance:none}. Clamp logic: Math.max(min, Math.min(max, value+delta)). Disable decrement when value===min, increment when value===max.

Design Decisions

Why buttons inside the input container instead of outside?
Embedding the buttons inside the container creates a single cohesive unit — the entire component feels like one form control. Buttons outside would look like separate actions rather than part of the input. The inner border-left/border-right between buttons and input maintains clear zones while keeping visual unity.
