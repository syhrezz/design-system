Wizard

A Wizard guides users through a multi-step process with a clear progress indicator, step validation, and linear navigation. Use it for onboarding, checkout, setup flows, and any task too complex to fit on a single form.

When to Use

Use Wizard when
  • A task requires 3 or more sequential steps
  • Each step depends on data from the previous one
  • You need to reduce cognitive load on long forms
  • Onboarding new users through a setup flow
  • Checkout or multi-step purchase flows
Avoid Wizard when
  • The task fits comfortably on one screen
  • Steps are optional or non-linear
  • Users need to jump back and forth freely
  • The process has only 2 steps — use a modal instead

Anatomy

1Step circle — shows step number; changes to checkmark when done
2Connector — line between steps; fills green as steps complete
3Step label — short name for each step, positioned below the circle
4Panel body — the active step content area
5Footer — Back / Next navigation and optional progress text

Accessibility

Add aria-label="Step X of N: Label" to each step circle. Wrap the step content in a role="region" with aria-live="polite" so screen readers announce step changes.

Best Practices

Do
  • Keep step labels short (1–2 words)
  • Validate each step before allowing Next
  • Always offer a Back button from step 2 onward
  • Show a summary/review step before final submission
  • Preserve form data when navigating back
Don't
  • Don't use more than 5–6 steps — split into separate pages
  • Don't lose user input on Back navigation
  • Don't hide the step count from users
  • Don't use a wizard for tasks that fit on one screen
  • Don't disable the Back button on the first step — hide it instead
