Accordion

Accordion allows sections of content to be expanded and collapsed independently. Used to progressively disclose information and reduce cognitive load when displaying multiple sections of related content.

When to Use

Use Accordion when
  • FAQ sections with many questions
  • Settings grouped by category
  • Long forms with logical sections
  • When content is optional — user chooses what to read
Don't use Accordion when
  • Content is short enough to display fully
  • Users need to compare sections side by side
  • Critical information that must always be visible
  • Only 1-2 sections — use Cards instead

Anatomy

1
Container
div.accordion. border:1px solid #E2E8F0, border-radius:12px, overflow:hidden. Contains all accordion items.
2
Item
div.accordion-item. border-bottom separates items. Last item has no border.
3
Trigger
button.accordion-trigger. Full width, flex space-between. Title left, chevron right. Hover: bg:#F8FAFC. Open: title color:#6366F1.
4
Chevron
16px SVG icon. Rotates 180deg on open. color:#94A3B8 default → #6366F1 open. transition:transform 0.2s.
5
Content panel
div.accordion-content. max-height:0 → max-height:400px transition (0.25s cubic-bezier). Inner div has padding:0 18px 16px.

Accessibility

RequirementImplementation
Trigger elementAlways a <button> — never a div with onclick.
aria-expanded"true" when open, "false" when closed on trigger button.
aria-controlsPoints to the content panel ID from the trigger button.
Content visibilityUse max-height:0 + overflow:hidden instead of display:none, so screen readers can still access content.

Best Practices

Do
Keep trigger labels concise (3-6 words). Open the most important section by default.
Users scan accordion triggers to decide what to open. Short labels scan faster.
Don't
Use accordion when content could be shown fully on the page without overwhelming the user.
Accordion hides information — only use when there is genuinely too much to show at once.

AI Implementation Guide

For AI coding assistants: Trigger is always a <button>. Content animates via max-height: 0 → max-height: 400px (not display:none). Transition: 0.25s cubic-bezier(0.4,0,0.2,1). Chevron uses transform:rotate(180deg) when open. Add aria-expanded to trigger. Use <button class="accordion-trigger"> — never a div. Container has border + border-radius:12px + overflow:hidden.

Design Decisions

Why max-height animation instead of height animation?
CSS cannot animate height:auto. max-height animation is the standard workaround — set max-height to a value larger than the content will ever be. The cubic-bezier easing makes the animation feel snappy and natural rather than mechanical.
