Timeline

Timeline displays a chronological sequence of events, activities, or status changes. Each item has a timestamp, a brief description, and an optional icon or category indicator. Used in activity feeds, audit logs, and status histories.

When to Use

Use Timeline when
  • Activity feed on a customer or user detail page
  • Audit log showing change history
  • Order/deal status progression
  • Notification history
Don't use Timeline when
  • Non-chronological data — use a table
  • Only 1-2 items — use a simple list
  • Data requires sorting/filtering — use a table with filter bar

Anatomy

1
Container
div.timeline. flex-direction:column. Relative positioning for the connecting line.
2
Dot indicator
32px circle. Soft colored background matching event type. Contains a 16px SVG icon. box-shadow ring in matching color.
3
Connecting line
2px wide, color:#F1F5F9, flex:1 height. Sits between consecutive dot indicators. Hidden on last item.
4
Body content
Title (13px fw-600) + description (12px #64748B) + timestamp (11px mono #94A3B8).

States

Timeline items are static display elements — no interactive states by default. If individual items are clickable (e.g., to expand details), add cursor:pointer and hover background (#F8FAFC) to the timeline-body. The connecting line is always present between items — it is removed only on the last item via :last-child selector.

Accessibility

RequirementImplementation
List semanticsUse <ol> for timeline container (ordered — chronological). Each item is <li>.
TimestampsUse <time datetime="ISO-date"> for timestamps so screen readers can announce them correctly.
Connecting lineDecorative — add aria-hidden="true" to the line element.

Best Practices

Do
Show newest events at the top. Use semantic type colors to indicate event importance.
Newest-first matches how users read feeds. Color provides at-a-glance context.
Don't
Show all 100+ events without pagination or "Show more".
Unlimited timelines overwhelm the page. Use maxItems with "Show more" for long histories.

Content Rules

Good title
Deal Closed
2 words, past tense, describes what happened.
Bad title
The deal was successfully closed by the sales representative
Too verbose. Timeline titles are 2-4 words.

AI Implementation Guide

For AI coding assistants: Each timeline-item has two children: .timeline-indicator (dot + line) and .timeline-body (title + desc + time). The connecting line is inside .timeline-indicator, flex:1 height, hidden on last item via :last-child. Dot size: 32px circle with type-specific bg color and icon. Timestamps use JetBrains Mono 11px. Newest event first (reverse chronological). Use <ol>+<li> for semantic list.

Design Decisions

Why dots with icons instead of plain colored dots?
Icons inside the dots provide a secondary visual cue beyond color — critical for accessibility since color alone cannot convey meaning. A phone icon immediately communicates "call" even before reading the text. This makes the timeline scannable at a glance.
