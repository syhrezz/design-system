Detail Page Template

The detail page template displays all information about a single entity (customer, prospect, user). It uses a tabbed layout with an info header, status badges, and separate tab panels for related data.

Wireframe

← Back to List
Entity Name + Status Badge
Edit · Delete
Main Info Card (name, contact, address, notes)
Stats Card (assigned rep, region, created date)
Overview
Activities
Transactions
Contacts
Tab Panel Content

Structure

/* Detail page structure */
1. Back button + Entity name + status badge + Edit/Delete actions
2. Info grid (grid-template-columns: 2fr 1fr):
   Left: Main info card (name, email, phone, address, notes)
   Right: Meta card (status, assigned rep, region, created date)
3. Tabs (underline variant): Overview | Activities | Transactions | Contacts
4. Tab panels (one per tab):
   - Overview: summary stats + recent activity preview
   - Activities: activity feed with .act-item rows
   - Transactions: table with amounts in mono font
   - Contacts: contact cards or list

AI Implementation Notes

For AI coding assistants: The back button uses an inline SVG left chevron + "Back to [List name]" text. The entity header row is flex space-between. Status badge sits inline next to the entity name. The info grid uses grid-template-columns:2fr 1fr. Tabs use the .tabs-ul underline variant. Each tab panel is a .tab-panel div that shows/hides via the .active class. Delete action always triggers a confirm modal.
