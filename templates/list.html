List Page Template

The list page is the standard template for browsable data collections: users, customers, prospects, products, and transactions. It combines a filter bar with a full CRUD table.

Wireframe

Page Header — h1 + subtitle + "Add [Item]" button
Search input (icon left)
Select: Role/Status
Select: Region
Table Wrap (toolbar + table + pagination)
Empty State (shown when data.length === 0)
Add/Edit Modal (hidden by default)
Delete Confirm Modal (hidden by default)

Structure

/* List page HTML structure */
<main>
  <div class="content">
    1. Page header
    <div class="page-header">...</div>

    2. Filter bar
    <div class="filter-bar">
      <div class="filter-item"><input class="input" type="search"/></div>
      <div class="filter-item"><select class="input">...</select></div>
    </div>

    3. Table
    <div class="table-wrap">
      <div class="table-toolbar">...</div>
      <table>...</table>
      <div class="table-pagination">...</div>
    </div>

    4. Empty state (conditional)
    <div class="empty-state" id="emptyState" style="display:none">...</div>
  </div>
</main>

5. Modals (outside main, inside body)
<div class="modal-overlay" id="addModal">...</div>
<div class="modal-overlay" id="deleteModal">...</div>

AI Implementation Notes

For AI coding assistants: The modals must be placed OUTSIDE the main content area but INSIDE the body tag — at the bottom of the page. The filter bar uses display:flex; gap:12px; flex-wrap:wrap; margin-bottom:24px. Filters trigger instantly on change event. The empty state div is shown/hidden based on whether the filtered data array is empty. Always include both an "add" modal and a "delete confirm" modal — even if the page only demonstrates read-only behavior.
